# Disposition note: Knowledge-as-Code — dormant, reserved as Canon Map's public home

Date: 2026-06-09
Author: Claude (Fable 5) session with Sam, from portfolio competitive review 2026-06-09
Status: Disposition decided with Sam. Record, not work order.

## Competitive finding (verified 2026-06-09)

Docs-as-code is a saturated, solved pattern: MkDocs/Material, Docusaurus, GitBook, git-backed wikis, CI-built knowledge bases. As a template, KaC adds opinionation and term-staking, not novelty. No investment justified on the current premise.

## Disposition

1. **Dormant.** Maintenance only. No content, promotion, or feature work on the docs-as-code template premise.
2. **Reserved future: public home of the Canon Map pattern.** The Canon Map scheme (typed source-of-truth model: kind / scope / canonical home / pointer convention / maintained by; pointer = 1-line link never paraphrase; snapshots carry verified dates; append-only handoffs with supersedes chains) passed its acceptance gate at v0.0.3 in the ob1-paice pilot on 2026-06-06. That pattern IS knowledge-as-code in a form nobody else has — drift-resistant knowledge architecture for multi-repo, multi-agent organizations — and is genuinely novel where the template is not.
3. **Promotion path** (do not start without Sam): Canon Map needs its second pilot (paice-foundation) and a Sam promotion decision first, per the 2026-06-06 session handoff ("zero tolerance for doing it wrong"; a third pilot is reasonable). Only after promotion to portfolio-wide practice does the public write-up belong here, replacing or absorbing the current template premise. knowledge-as-code.com is the staked term waiting for that content.

## Revisit triggers

- Canon Map promoted to portfolio-wide practice (post second/third pilot, Sam decision).
- External interest in the Canon Map write-up (would accelerate the promotion question, not bypass it).
